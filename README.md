# Vigilant — Website Monitoring

Vigilant é uma plataforma de **monitoramento de sites e serviços**, criada para ajudar equipes a acompanhar a disponibilidade e o desempenho de sua infraestrutura em tempo real.

O objetivo do produto é simples: **avisar antes que o cliente perceba que algo caiu**. Para isso, o sistema centraliza monitoramento, incidentes, relatórios e comunicação com usuários em um único painel.

## Para que serve

- **Monitorar a saúde de sites, APIs e serviços**, identificando quedas e lentidão assim que ocorrem.
- **Gerenciar incidentes**, dando visibilidade ao time sobre o que está fora do ar e por quê.
- **Publicar uma página de status pública**, para manter clientes informados sobre a disponibilidade dos serviços.
- **Gerar relatórios** de uptime e desempenho ao longo do tempo.
- **Organizar times e integrações**, conectando o monitoramento a outras ferramentas usadas no dia a dia.

## Funcionalidades da aplicação

| Área | Descrição |
|---|---|
| **Dashboard** | Visão geral do status de todos os serviços monitorados. |
| **Monitoramentos** | Cadastro e acompanhamento individual de cada serviço monitorado. |
| **Incidentes** | Histórico e gestão de quedas e problemas identificados. |
| **Status Page** | Página pública de status para comunicação com usuários finais. |
| **Relatórios** | Métricas de uptime e desempenho. |
| **Integrações** | Conexão com outras ferramentas e serviços. |
| **Equipe** | Gestão dos membros com acesso à plataforma. |
| **Configurações** | Ajustes gerais da conta e do sistema. |

Além da aplicação em si, o projeto conta com uma **landing page** (apresentação do produto) e um **fluxo de autenticação** (login e criação de conta) para acesso ao painel.

## Stack

Construído com [Next.js](https://nextjs.org) (App Router) e React, estilizado com Tailwind CSS e animado com Framer Motion.
